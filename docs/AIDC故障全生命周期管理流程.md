# AIDC 故障全生命周期管理流程（SOP V1.0）

> 覆盖故障发现、分级研判、应急响应、隔离定位、修复执行、恢复验证、复盘归档全流程。每一步操作均要求明确确认。
>
> 撰写人：孟希东（测试）

---

## 故障分级定义

| 等级 | 定义 | 典型场景 | 响应时效 | 升级对象 |
|------|------|----------|----------|----------|
| **P0 灾难** | 整集群不可用/数据丢失 | 机柜断电、液冷主管泄漏、级联故障 | 5 分钟 | CTO/VP |
| **P1 严重** | 训练中断/部分不可用 | 多卡 GPU 故障、NVLink 中断、CDU 故障 | 15 分钟 | 运维总监 |
| **P2 一般** | 性能降级但可用 | 单卡 ECC、GPU 降频、单 PSU 故障 | 30 分钟 | 运维经理 |
| **P3 轻微** | 不影响当前业务 | 阈值预警、非关键指标异常 | 4 小时 | 值班工程师 |

## Phase 1 · 故障发现与告警确认

### 1.1 接收告警信号
- 确认告警来源（AIOps/Prometheus/DCIM/液冷控制器/人工巡检）
- 记录告警触发时间（精确到秒）、告警 ID、内容摘要
- 🔴 **必须确认**：值班工程师 **2 分钟内** 确认接收，超时自动升级

### 1.2 告警真实性验证
- 至少两个独立数据源交叉验证（nvidia-smi + Prometheus + DCIM）
- GPU 告警：`nvidia-smi -q -i [GPU_ID]`
- 液冷告警：CDU 控制面板流量/压力/温度
- 🔴 **必须确认**：标注"已验证为真实故障"或"确认为误报"

### 1.3 采集故障现场快照
- GPU：`nvidia-smi -q`、`dmesg | grep -i gpu`、XID 错误日志
- 液冷：CDU 参数截图
- 供电：PSU 状态、母线温度
- 网络：`nvidia-smi nvlink -e`、IB 端口状态
- 🔴 **严禁在快照采集完成前执行任何修复操作**

### 1.4 创建故障工单
- 必填：故障标题、发现时间、告警来源、故障对象、初步现象

## Phase 2 · 故障分级与影响研判

### 2.1 故障域判定
- 判定域：GPU 算力/液冷/供电/网络/存储/基础设施
- 跨域故障需同时通知所有相关域负责人

### 2.2 影响范围评估
- 受影响任务数、GPU 数量占集群百分比
- 扩散风险评估、冗余保护状态
- 最近 checkpoint 时间、数据丢失风险

### 2.3 故障定级
- 对照分级表确定 P0/P1/P2/P3
- P0/P1 必须**电话通知**升级对象（非仅消息）
- P0 启动应急作战室（War Room）

### 2.4 通知相关方
- 通知受影响业务方技术负责人
- 液冷/供电故障通知现场工程师和供应商
- GPU 硬件故障通知 NVIDIA TAM

---

## Phase 3 · 应急响应与任务保护

### 3.1 紧急保存 Checkpoint
- 确认最近 checkpoint 时间和 step 号
- 距上次 >1 小时或 >1000 steps → 评估紧急保存

### 3.2 训练任务迁移或暂停
- 有空闲 GPU → 调度系统迁移任务，从 checkpoint 恢复
- 无法迁移 → graceful shutdown（严禁直接 kill -9）

### 3.3 故障扩散防护
- GPU：drain node，标记不可调度
- 液冷泄漏：**立即关闭管路阀门 → 断电 → 疏散人员**
- 供电异常：监控其余 PSU 负载，设置加强告警
- 网络：隔离故障端口，确认流量切换至冗余路径

---

## Phase 4 · 故障隔离与根因定位

### 4.1 分域深度诊断

**GPU 故障**：XID 错误码 → ECC 计数 → Retired Pages → `dcgmi diag -r 3`

**液冷故障**：CDU 日志 → 水质报告 → 管路外观 → 泵组振动

**供电故障**：PSU 电压/电流/温度 → 母线红外热成像 → UPS 自检

**网络故障**：NVLink 错误计数 → IB SymbolError → 光模块收发光功率

### 4.2 关联分析与根因推断
- 按时间线排列异常事件，确认因果顺序
- 对照历史案例库
- 无法定位 → 联系设备厂商技术支持

### 4.3 根因确认评审
- P0/P1：召集专家评审会议
- P2/P3：运维经理审核确认

---

## Phase 5 · 修复方案制定与执行

### 5.1 制定修复方案
- 操作步骤（精确到每条命令）+ 风险评估 + 回滚方案 + 备件 + 预计耗时

### 5.2 修复方案审批
- P0/P1：运维总监审批 | P2：运维经理 | P3：高级工程师
- **未经审批严禁执行修复操作**

### 5.3 修复前预检
- 故障设备已隔离，不影响同机架健康设备
- 液冷/供电操作：**双人作业**，确认阀门关闭/回路断开
- 操作人员和复核人员双签确认

### 5.4 执行修复
- **逐步执行，每一步确认结果正常后再进行下一步**
- 实时记录每步执行时间和结果
- 异常则暂停，评估是否回滚

### 5.5 修复后即时检查
- GPU：`nvidia-smi` → ECC 清零 → `dcgmi diag -r 2`
- 液冷：压力恢复 → 流量正常 → 无泄漏 → 温度回落
- 供电：电压电流正常 → 母线温度正常 → 冗余恢复

---

## Phase 6 · 恢复验证与业务回切

### 6.1 功能性验证
- `dcgmi diag -r 3` + gpu-burn + NCCL allreduce
- 液冷/供电持续观察 15-30 分钟

### 6.2 压力测试验证
- 接近生产负载运行 **≥1 小时**，全程无告警

### 6.3 解除隔离
- 节点从 drain → schedulable → 确认 Ready

### 6.4 业务回切
- 先调度低优先级任务 → 稳定后逐步回切高优先级
- 业务方确认任务恢复正常

### 6.5 持续观察期

| 故障等级 | 观察期 |
|----------|--------|
| P0/P1 | 72 小时 |
| P2 | 48 小时 |
| P3 | 24 小时 |

- 设置加强监控，每日检查关键指标
- 观察期内复发 → 重新进入 Phase 4

---

## Phase 7 · 复盘归档与持续改进

### 7.1 复盘
- P0/P1：修复后 3 工作日内召开复盘会议
- P2/P3：书面复盘

### 7.2 故障报告
- 包含：概述、影响范围、详细时间线、根因、修复措施、改进项

### 7.3 知识库与监控优化
- 更新故障案例库、运维手册/SOP、告警规则、AIOps 模型

### 7.4 改进项跟踪
- 每项改进创建跟踪工单，每周回顾直至闭环

### 7.5 工单关闭
- 计算 MTTD / MTTR / 业务影响时长，纳入月度统计
