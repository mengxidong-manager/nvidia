## 1. 六大运维域

### 1.1 GPU 算力运维

NVL72 机架中 72 颗 GPU 通过 NVLink 互联为整体，任何一颗故障都可能中断整个训练任务。

- GPU 健康监控：温度、ECC 错误、降频事件
- 训练任务调度与故障自动迁移
- NVLink 链路带宽和错误率监控
- RAS 引擎不停机预防性维护

### 1.2 液冷系统运维

- CDU 流量、压力、温度监控
- 冷却液水质检测（pH/电导率/颗粒物）
- 快接头和管路泄漏检测
- 供回水温差（ΔT）异常告警

### 1.3 高密供电运维

- 8 个电源架、48 个 PSU 模块状态监控
- 铜母线温度监控
- UPS / PDU 巡检
- 未来 800V HVDC 新设备运维

### 1.4 高速网络运维

- NVLink 交换托盘链路状态
- InfiniBand / Spectrum-X 拥塞与丢包
- ConnectX SuperNIC 与 BlueField DPU 监控

### 1.5 存储运维

- 分布式存储 IOPS 与吞吐监控
- Checkpoint 写入性能
- 数据集预处理管道可用性

### 1.6 基础设施运维

- 液冷管路漏液传感器
- 高精度温湿度监控
- 消防系统与物理安全
- 能效 PUE 实时追踪

## 2. AIOps 智能运维体系

| 能力层级 | 指标 | 说明 |
|----------|------|------|
| **监控层** | 秒级 | 基于业务视图自动创建监控任务 |
| **排障层** | <1min | 实时识别异常，自动分析根因，推荐修复方案 |
| **变更层** | 仿真 | 意图识别、方案推荐、下发前仿真评估、按需回滚 |
| **调优层** | 自适应 | 根据流量模型自动调参，建立动态基线 |

> 核心理念：从被动响应到主动预防。MTTD 从分钟级降至秒级，MTTR 降低 70%+。

## 3. 典型工作流

### 3.1 日常巡检流程

1. GPU 集群健康检查
2. 液冷系统巡检
3. 供电系统巡检
4. 网络链路巡检
5. 环境监控确认
6. 生成巡检报告

### 3.2 故障处理流程

1. AI 预测/告警触发
2. 自动根因分析
3. 判断是否可自愈
4. 自愈执行或工单派发
5. 执行修复 → 验证恢复
6. 知识沉淀

### 3.3 变更管理流程

1. 变更申请 → AI 仿真评估
2. 审批 → 灰度执行
3. 自动验证 → 全量推送

## 4. 与传统 DC 对比

| 维度 | 传统 DC | AIDC |
|------|---------|------|
| 核心资产 | 服务器 CPU | GPU 集群（机架级） |
| 散热 | 风冷为主 | 100% 液冷 |
| 单柜功率 | 5-15 kW | 100-600 kW |
| 故障影响 | 单台服务器 | 整个训练任务中断 |
| 运维技能 | 网络+系统 | +液冷+高压配电+GPU |
| 运维模式 | 人工+脚本 | AIOps+自愈 |
| 核心 KPI | Uptime | **Tokens per Watt** |
