## 1. 架构总览

Blackwell Ultra（B300）是 Blackwell（B200）的演进版，优化重心从训练转向推理——更多 FP4 算力、更大显存、2 倍注意力加速。GB300 NVL72 是其机架级部署形态。

### 产品矩阵

| 产品 | 配置 | 定位 |
|------|------|------|
| B300 GPU | 单颗 Blackwell Ultra 芯片 | GPU 本体，用于各种系统集成 |
| HGX B300 | 8 × B300 + x86 CPU | 服务器级，OEM 集成（Dell/HPE/Lenovo/Supermicro） |
| DGX B300 | 8 × B300，NVIDIA 整机 | 企业 AI 工厂，192 PFLOPS 推理 |
| **GB300 NVL72** | **72 × B300 + 36 × Grace CPU** | **机架级统一加速器，1.1 EFLOPS** |

### Grace Blackwell Ultra Superchip

GB300 NVL72 的基本计算单元。与 GB200 不同，GB300 采用新的计算托盘设计：单个载板上集成 2 颗 Grace CPU 和 4 颗 B300 GPU（而非 GB200 的两块独立 1C:2G 板）。CPU、GPU 和 NIC 均采用插槽式设计，可独立更换。

**GB300 计算托盘布局（从上到下）**：NVLink 连接器 → 4× B300 GPU → 2× Grace CPU → 4× CX-8 NIC

**关键设计变化（vs GB200）**：GB300 将 ConnectX-8 NIC 集成到计算托盘上，并使用外部 PCIe 交换机，解决了 GB200 中 RDMA 流量需要穿越 Grace NoC 再跨 NVLink-C2C 到达 GPU 的性能瓶颈。所有组件均可插拔独立更换，提升可维护性。

## 2. 计算芯片

B300 是全新硅片（非 B200 改版），针对推理场景深度优化：FP4 算力翻倍、SFU 翻倍解决注意力瓶颈、HBM3e 12-high 堆叠提升容量。

### B300 GPU（Blackwell Ultra）

全新硅片设计，针对推理/推理时间缩放（test-time scaling）场景优化。FP64 性能大幅削减以换取更高的 FP4 推理算力。

- 制程：TSMC 4NP · 双芯片
- FP4 Dense：15 PFLOPS
- FP8：5 PFLOPS
- FP16/BF16：2.5 PFLOPS
- HBM3e：288 GB（12-high）
- 显存带宽：8 TB/s
- NVLink：1.8 TB/s 双向
- TDP：1,400W

### Grace CPU（ARM）

NVIDIA 自研 Arm CPU，通过 NVLink-C2C 与 B300 GPU 实现内存一致性连接。

- 架构：Arm Neoverse V2
- 核心：72 核
- 内存：LPDDR5X
- C2C 带宽：900 GB/s
- 定位：主机 CPU + 统一内存

### B300 vs B200 关键差异

| 指标 | B200 | B300 | 提升 |
|------|------|------|------|
| FP4 Dense | 9 PFLOPS | **15 PFLOPS** | **+66.7%** |
| FP8 | 4.5 PFLOPS | 5 PFLOPS | +11.1% |
| FP16/BF16 | 2.25 PFLOPS | 2.5 PFLOPS | +11.1% |
| HBM 容量 | 192 GB | **288 GB** | **+50%** |
| HBM 带宽 | 8 TB/s | 8 TB/s | 持平 |
| HBM 堆叠 | 8-high | **12-high** | +50% |
| SFU 性能 | 5 TeraExp/s | **10.7 TeraExp/s** | **+114%** |
| 注意力加速 | 1× | **2×** | **翻倍** |
| FP64 | 37 TFLOPS | 1.25 TFLOPS | -96.6%（有意削减） |
| TDP | 1,000W | 1,400W | +40% |

> **SFU 翻倍解决 Softmax 瓶颈**：B200 将 SM 数量从 B100 的 132 减少到 80，导致 SFU（特殊功能单元）性能不足，Softmax 计算成为注意力层的瓶颈。B300 将 SFU 从 5 TeraExp/s 提升到 10.7 TeraExp/s，翻倍解决了这个问题。这对长上下文推理和推理时间缩放至关重要。

## 3. 散热系统

1,400W 单卡 TDP 意味着风冷完全不可行。所有 B300 系统均强制液冷。

**100% 液冷**：DGX B300 和 GB300 NVL72 均配备集成液冷系统。HGX B300 基板需要 OEM 提供液冷方案。风冷无法处理 B300 级别的热密度。Supermicro DLC-2 方案可捕获 98% 的废热。

**机架级散热**：GB300 NVL72 机架功耗 132-140kW，采用直接液冷（DLC）方案。CDU（冷却液分配单元）可选机架内或行级部署，配合 RDHx 和 L2A sidecar 冷却。

> **vs Vera Rubin**：GB300 仍使用传统液冷架构，需要冷水机组。Vera Rubin 进化到 45°C 温水冷却，不再需要冷水机，是下一代的显著升级。

## 4. 供电架构

单机柜 132-140kW，采用与 GB200 NVL72 相同的 Oberon 机架供电架构，增加了功率平滑技术。

### 供电链路

```
电网(10-35kV AC) → 变压器(→480V AC) → UPS → PDU → 电源架(→54V DC) → 母线(54V·2500A+) → VRM(→~0.7V)
```

**电源架配置**：8 个电源架（48U 机架中占 8U），每架 6 × 5.5kW PSU。480V 三相 AC → 54V DC，效率约 97%。电源架液冷散热。总供电能力 264kW，覆盖计算和网络负载。

**功率平滑技术**：GB300 引入能量存储和消耗机制，在训练工作负载中稳定功率波动。同步 GPU 负载爬升时，可削减峰值功率尖峰最高达 30%，降低对设施层面电力系统的冲击。

> **设施规划要点**：到 2026 年，部署 GB300 最难的部分往往不是订购 GPU，而是准备设施——液冷管路、电力分配、冗余设计、甚至机架的结构承重都需要提前规划。每个机架 132-140kW 的功率密度远超传统数据中心的承载能力。

## 5. 内存与存储

288 GB HBM3e（12-high 堆叠）是 B300 最重要的升级之一，使得 DeepSeek-R1（671B MoE）等巨型模型可以用更少的 GPU 运行推理。

**HBM3e 12-High（288 GB）**：相比 B200 的 192 GB（8-high）增加 50%。12 层堆叠的 HBM3e 在相同封装面积内提供更大容量。每颗 GPU 8 TB/s 带宽。整个 NVL72 机架 HBM 总容量约 20.7 TB。

**LPDDR5X（Grace CPU）**：每颗 Grace CPU 配备 LPDDR5X 内存。通过 NVLink-C2C 以 900 GB/s 带宽与 GPU 一致性连接，应用可将 CPU 和 GPU 内存视为统一地址空间。

**NVLink 统一内存**：72 颗 GPU 通过 NVLink 5 全互联，整个机架的 20.7 TB HBM 和 CPU 内存构成一个巨大的统一内存池。这使得超大模型无需复杂的张量并行就能在机架内运行。

### 内存层次

| 层级 | 介质 | 容量 / GPU | 带宽 | 用途 |
|------|------|-----------|------|------|
| L1/L2 Cache | SRAM | 数 MB | TB/s 级 | 寄存器/缓存 |
| HBM3e | HBM3e 12-high | 288 GB | 8 TB/s | 模型权重 / KV Cache |
| CPU LPDDR5X | LPDDR5X | 按配置 | 546 GB/s | 预处理 / 溢出 |
| NVLink 域 | 全互联 HBM | ~20.7 TB（机架） | 130 TB/s | 分布式模型推理 |
| 网络存储 | NVMe / 分布式 | PB 级 | 变化 | Checkpoint / 数据集 |

> **288 GB 的实际意义**：DeepSeek-R1（671B MoE）可以用更少的 GPU 运行推理，降低张量并行开销。在 MLPerf Inference v6.0（2026年4月）中，Blackwell Ultra 在 DeepSeek-R1 上实现了 250 万 tokens/秒的吞吐量，比首次提交高 2.7 倍。

## 6. 网络互联

双层网络：NVLink 5 机架内全互联（scale-up） + Spectrum-X / Quantum-X800 跨机架通信（scale-out）。

### Scale-Up：NVLink 5（机架内全互联，130 TB/s）

第五代 NVLink 为每颗 GPU 提供 1.8 TB/s 双向带宽（900 GB/s 单向）。72 颗 GPU 通过 NVLink Switch 全互联，机架 bisection 带宽 130 TB/s。整个机架表现为一个统一的分布式加速器，而非 72 个独立 GPU 的集群。

### Scale-Out：双模可选

**Spectrum-X Ethernet**：NVIDIA AI 优化以太网。800 Gb/s / GPU，支持 RoCE（RDMA over Converged Ethernet）。适合推理为主的部署，运维团队熟悉以太网的场景。

- 带宽：800 Gb/s / GPU
- 协议：RoCE v2
- 扩展：数万 GPU

**Quantum-X800 InfiniBand**：传统 AI 训练首选。Microsoft Azure 描述为 800 Gbps / GPU 的非阻塞架构，可扩展到数万 GPU。硬件级集合通信加速。

- 带宽：800 Gb/s / GPU
- 优势：AllReduce 硬件卸载
- 扩展：数万 GPU

### ConnectX-8 SuperNIC（Scale-out 网络接口，800 Gb/s）

每颗 GPU 配备 ConnectX-8 SuperNIC，提供 800 Gb/s 的 scale-out 带宽。在 GB300 中，CX-8 直接集成在计算托盘上（B200 中是挂在 Grace CPU 后面），通过外部 PCIe 交换机直连 GPU，消除了 RDMA 流量穿越 Grace NoC 的瓶颈。支持 InfiniBand 和 800G Ethernet（CX-8 仅 IB 支持 800G，以太网需 CX-9）。

### 互联带宽代际对比
