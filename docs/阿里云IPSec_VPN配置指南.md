# 阿里云 IPSec VPN 配置指南

> 从零配置阿里云 IPSec-VPN，实现本地数据中心与 VPC 加密互通。涵盖网络规划、资源创建、参数配置、路由设置、对端设备配置及故障排查。
>
> 撰写人：孟希东（测试）

---

## 架构概览

```
本地数据中心(172.16.0.0/16) → 本地网关(公网IP) ⇄ IPSec隧道 ⇄ 阿里云VPN网关(公网IP) → VPC(192.168.0.0/16)
```

| 组件 | 说明 |
|------|------|
| **VPN 网关** | 阿里云侧出入口，部署在 VPC 内，绑定公网 IP |
| **用户网关** | 本地网关设备公网 IP 注册到阿里云 |
| **IPSec 连接** | 定义隧道参数（IKE/IPSec 加密策略、路由模式） |
| **路由配置** | VPN 网关侧 + VPC 侧 + 本地设备侧三层路由 |

---

## Step 1 · 前置准备

- [ ] 确认 VPC 网段（如 192.168.0.0/16），与本地网段**不能重叠**
- [ ] 确认本地数据中心网段（如 172.16.0.0/16）
- [ ] 确认本地网关设备固定公网 IP
- [ ] 确认 VPN 带宽需求（10M/100M/200M/500M/1000M）
- [ ] 与本地网络管理员协商 IKE/IPSec 参数
- [ ] RAM 权限：AliyunVPNGatewayFullAccess + AliyunVPCFullAccess

---

## Step 2 · 创建 VPN 网关

VPN 控制台 → 创建 VPN 网关：

| 参数 | 建议值 |
|------|--------|
| 地域 | 与 VPC 同地域 |
| VPC | 选择目标 VPC |
| 带宽规格 | 按业务需求 |
| 网络类型 | **公网** |
| IPSec-VPN | **开启** |
| 隧道模式 | **双隧道（推荐，高可用）** |

> 创建完成后记录 VPN 网关公网 IP（双隧道分配两个 IP）。

---

## Step 3 · 创建用户网关

| 参数 | 说明 |
|------|------|
| IP 地址 | 本地网关设备公网 IP |
| ASN | 本地 BGP 自治系统号（启用 BGP 时填写） |

> 双隧道模式：两个公网出口 → 创建两个用户网关；一个公网 IP → 创建一个即可。

---

## Step 4 · 创建 IPSec 连接（核心步骤）

### 路由模式选择

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **目的路由**（推荐） | 基于目的 IP 转发，无需填本端/对端网段 | BGP 或静态路由 |
| 感兴趣流 | 基于源+目的 IP，需配置本端/对端网段 | 仅特定网段互通 |

### IKE 配置（第一阶段）

| 参数 | 推荐值 |
|------|--------|
| IKE 版本 | **IKEv2** |
| 加密算法 | **aes256** |
| 认证算法 | **sha256** |
| DH 分组 | **group14**（2048-bit） |
| SA 生命周期 | 86400 秒（24 小时） |
| 预共享密钥 | 强密码 ≥16 位混合字符，**两端必须一致** |

### IPSec 配置（第二阶段）

| 参数 | 推荐值 |
|------|--------|
| 加密算法 | **aes256** |
| 认证算法 | **sha256** |
| DH 分组（PFS） | **group14** |
| SA 生命周期 | 3600 秒（1 小时） |
| DPD | **开启** |
| NAT-T | 开启（如有 NAT） |

### 感兴趣流配置（仅感兴趣流模式）

| 参数 | 示例 |
|------|------|
| 本端网段 | 192.168.0.0/16（VPC CIDR） |
| 对端网段 | 172.16.0.0/16（IDC CIDR） |

> 创建完成后点击**"下载对端配置"**，用于配置本地设备。

---

## Step 5 · 配置路由

### 三层路由缺一不可

1. **VPN 网关路由**：目标网段 172.16.0.0/16 → 下一跳 IPSec 连接
2. **VPC 路由表**：发布 VPN 网关路由到 VPC（勾选"已发布"）
3. **本地设备路由**：目标 192.168.0.0/16 → 下一跳 VPN 隧道接口

### BGP 动态路由（可选）

| 参数 | 示例 |
|------|------|
| 本端 ASN | 45104（默认） |
| 对端 ASN | 65001（私有 ASN） |
| 隧道网段 | 169.254.10.0/30 |

---

## Step 6 · 配置本地网关设备

### 放通端口和协议

```bash
iptables -I INPUT -p udp --dport 500 -j ACCEPT    # IKE
iptables -I INPUT -p udp --dport 4500 -j ACCEPT   # NAT-T
iptables -I INPUT -p 50 -j ACCEPT                 # ESP
```

### 开启 IP 转发（Linux）

```bash
echo "net.ipv4.ip_forward = 1" >> /etc/sysctl.conf
sysctl -p
```

### 设备配置参考

阿里云官方文档提供 Cisco ASA、华为 USG、Juniper SRX、strongSwan、FortiGate、pfSense 等设备示例。

---

## Step 7 · 连通性验证

1. 阿里云控制台 → IPSec 连接 → IKE SA: 已连接 ✓ / IPSec SA: 已连接 ✓
2. ECS 端 ping 本地：`ping 172.16.1.100`
3. 本地端 ping ECS：`ping 192.168.1.100`
4. 单向通 → 检查安全组和本地 ACL

## 故障排查

### IKE 第一阶段失败

| 原因 | 排查 |
|------|------|
| PSK 不一致 | 对比两端，注意空格和大小写 |
| IKE 版本不一致 | 两端均为 IKEv1 或 IKEv2 |
| 加密/认证算法不匹配 | 逐项对比 |
| DH 分组不匹配 | 必须相同 |
| 防火墙阻断 | 检查 UDP 500/4500 |

### IKE 成功但 IPSec 失败

| 原因 | 排查 |
|------|------|
| 第二阶段算法不匹配 | 对比加密/认证/PFS |
| 感兴趣流不匹配 | 阿里云本端 = 对端远端 |
| ESP 协议被阻断 | 检查 IP 50 |

### 隧道通但 Ping 不通

| 原因 | 排查 |
|------|------|
| 路由缺失 | VPN 网关/VPC/本地三层路由 |
| 路由未发布 | 确认"已发布"到 VPC |
| 安全组阻断 | ECS 入方向允许本地网段 |
| 本地 ACL | 允许 VPC 网段入站 |

## 推荐安全组合

| 安全等级 | 加密 | 认证 | DH | 适用 |
|----------|------|------|-----|------|
| 标准 | aes128 | sha1 | group2 | 一般业务 |
| **推荐** | **aes256** | **sha256** | **group14** | **生产环境** |
| 高安全 | aes256 | sha512 | group24 | 金融/合规 |
